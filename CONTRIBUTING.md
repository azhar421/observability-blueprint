# Contributing to Observability Blueprint

Thank you for your interest in contributing to this project.

The goal of this repository is to provide a production-inspired learning platform for observability technologies including Prometheus, Grafana, Loki, Tempo, OpenTelemetry, Docker, Kubernetes, Ansible, and Terraform.

## Ways to Contribute

You can contribute by:

- Improving documentation
- Fixing typos
- Adding deployment examples
- Creating troubleshooting guides
- Adding Grafana dashboards
- Writing automation scripts
- Reporting bugs
- Suggesting new features

## Pull Request Process

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Test your changes.
5. Submit a Pull Request with a clear description.

## Code Quality

Please ensure:

- Clear documentation
- Proper formatting
- Meaningful commit messages
- Production-inspired examples
- No confidential or proprietary information

Thank you for helping improve this project.
